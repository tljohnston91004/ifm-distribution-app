Full detailed consolidated version
Status: Full detailed consolidated working document. All approved addenda have been merged into the main body. Superseded wording has been removed.
1. Purpose and Data Principle
This document defines the data backbone for IFM-Distribution V1.5. Cursor and the developer should use it to build database tables, import mapping, validation, source tracking, confidence logic, and calculation input snapshots.
Locked Hard Rules
IFM is source-agnostic. QuickBooks and Keystroke are example/test sources, not required systems.
IFM maps source data into standard IFM data domains before calculations run.
IFM does not create, send, modify, cancel, pay, borrow, or factor automatically in V1.5.
Funding approval is separate from purchase approval and separate from official PO action.
Buyer action in Keystroke or another purchasing system happens outside IFM in V1.5.
LOC and factoring are supplemental funding, not operating cash.
Vendor deals, discounts, free freight, and extended terms are not automatically good.
RSE identifies replenishment need and inventory classification; IFM allocates funding priority.
Blank or missing data must not be treated as zero unless confirmed none exist.
Any file intended for Keystroke upload or re-import must be tab-delimited and end in .txt.
CSV/XLSX may be used for review, mapping, QA, internal analysis, or customer review, but not as Keystroke import-ready files.
Customer-facing output must protect sensitive internal, financing, AR, AP, and reviewer detail unless approved for display.
2. Source-Agnostic Data Model Rule
IFM separates IFM standard domains from source systems. A QuickBooks AP Aging report, Sage AP report, NetSuite Open Bills export, or manual AP schedule can all feed the same ap_items table if required fields are mapped and validated.
