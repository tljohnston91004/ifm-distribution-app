1. Purpose
This document defines IFM screens, workflow modules, user roles, warning banners, next actions, approvals, output workflow, and build priority. Screen means page, tab, panel, section, modal, detail view, or workflow step; Cursor should not build 33 disconnected pages unless necessary.
Locked Hard Rules
IFM is source-agnostic. QuickBooks and Keystroke are example/test sources, not required systems.
IFM maps source data into standard IFM data domains before calculations run.
IFM does not create, send, modify, cancel, pay, borrow, or factor automatically in V1.5.
Funding approval is separate from purchase approval and separate from official PO action.
Buyer action in Keystroke or another purchasing system happens outside IFM in V1.5.
LOC and factoring are supplemental funding, not operating cash.
Vendor deals, discounts, free freight, and extended terms are not automatically good.
RSE identifies replenishment need and inventory classification; IFM allocates funding priority.
Blank or missing data must not be treated as zero unless confirmed none exist.
Any file intended for Keystroke upload or re-import must be tab-delimited and end in .txt.
CSV/XLSX may be used for review, mapping, QA, internal analysis, or customer review, but not as Keystroke import-ready files.
Customer-facing output must protect sensitive internal, financing, AR, AP, and reviewer detail unless approved for display.

8. Warning Banners and Next Best Action
Warning condition
Suggested banner
Negative core funding
Core operating cash does not support new inventory purchases without supplemental funding or management override.
LOC/factoring required
This purchase depends on supplemental funding and requires approval.
Reserve override
This purchase would use cash below protected reserve. Owner approval required.
Stale data
One or more key sources are stale and may affect confidence.
Missing source of truth
Conflicting data exists. Select source of truth before final output.
Vendor concentration
This vendor consumes a high percentage of available inventory funding.
Keystroke export format failure
Keystroke imports must be tab-delimited .txt.
Approved run locked
Create a revision before changing approved data.
Buyer override pending
Buyer request requires reviewer/owner decision.
Data gap blocks decision
A material gap blocks one or more purchase decisions.
